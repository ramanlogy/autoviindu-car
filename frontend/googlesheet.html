Google Sheets Integration

Connect Any Form to
Google Sheets

A single reusable pattern for all AutoViindu service pages. Add once, adapt per form — no backend, no database, no cost.

15 min setup per sheet No server required Reusable pattern Works on all form pages
How It Works
The form sends a JSON payload to a Google Apps Script Web App, which writes a row into a Google Sheet. The browser uses mode:'no-cors' so no response is received — but the data lands in the sheet reliably. The success screen shows immediately based on the submission attempt, not the response.

Step 2 — The Apps Script (paste once per sheet)
2
Open Apps Script and paste the handler
Extensions → Apps Script → delete existing code → paste below

In your Google Sheet go to Extensions → Apps Script. Delete any existing code and paste this. The doPost function runs every time the form submits.

Apps Script (Google) 
// Parts & Accessories — Apps Script handler
// Paste into Extensions → Apps Script → Save → Deploy

function doPost(e) {
  try {
    var data    = JSON.parse(e.postData.contents);
    var sheet   = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();

    // Append one row — order must match your column headers in Row 1
    sheet.appendRow([
      data.timestamp,
      data.ref,
      data.requestType,
      data.itemType,
      data.itemDescription,
      data.partCode,
      data.quantity,
      data.installationNeeded,
      data.vehicleBrand,
      data.vehicleModel,
      data.variant,
      data.year,
      data.fuelType,
      data.fullName,
      data.mobile,
      data.deliveryAddress,
      data.email,
      data.deliveryMethod
    ]);

    return ContentService
      .createTextOutput(JSON.stringify({ status: 'ok', ref: data.ref }))
      .setMimeType(ContentService.MimeType.JSON);

  } catch(err) {
    return ContentService
      .createTextOutput(JSON.stringify({ status: 'error', message: err.toString() }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
Key order matters. The array in appendRow() must be in the same order as your Row 1 headers. If you add or remove a field from the form, update both the sheet header AND the appendRow array together.
Step 3 — Deploy as Web App
3
Deploy and copy the Web App URL
Do this once per script — redeploy when you change the script
  • In Apps Script, click Deploy → New Deployment
  • Click the gear icon next to "Type" and select Web App
  • Set "Execute as" → Me
  • Set "Who has access" → Anyone (required for the form to post without login)
  • Click Deploy → authorize when prompted → copy the Web App URL
Redeploying after changes: If you edit the script, you must create a New Deployment (not update the existing one) to get a new URL — or use Manage Deployments → Edit and set version to "New version". The old URL will stop working if you just save without redeploying.
Step 4 — The reusable JS pattern (same on every page)
4
The submit function — copy this pattern into every form page
Only the URL and payload object change between pages

This is the complete, reusable submit block. Paste it into any form page. Change two things only: the SHEETS_URL constant and the payload object keys to match that page's fields.

JavaScript — paste into <script> on any form page 
/* ─── 1. PASTE YOUR WEB APP URL HERE ─────────────────────────────── */
const SHEETS_URL = 'https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec';
/* ──────────────────────────────────────────────────────────────────── */

/**
 * generateRef()
 * Creates a short reference number like AV-284931.
 * Same function on every page — no change needed.
 */
function generateRef() {
  return 'AV-' + Date.now().toString().slice(-6);
}

/**
 * submitToSheets(payload)
 * Sends any object to Google Sheets.
 * Same function on every page — no change needed.
 */
async function submitToSheets(payload) {
  if (!SHEETS_URL || SHEETS_URL.includes('YOUR_DEPLOYMENT_ID')) {
    // Demo mode — log to console, don't error
    console.log('[DEMO] Payload:', payload);
    await new Promise(r => setTimeout(r, 700));
    return;
  }
  await fetch(SHEETS_URL, {
    method: 'POST',
    mode:   'no-cors',      // required — no response body returned
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload)
  });
}

/**
 * handleSubmit(event)
 * ─── CHANGE THIS SECTION per form page ───────────────────────────
 * Collect your form values, build the payload, call submitToSheets.
 */
async function handleSubmit(event) {
  event.preventDefault();

  // Basic validation — add your own checks here
  const name    = document.getElementById('fullName')?.value.trim();
  const mobile  = document.getElementById('mobile')?.value.trim();
  if (!name || !mobile) {
    alert('Please fill all required fields.');
    return;
  }

  // Disable button, show loading state
  const btn = event.target.querySelector('.form-submit');
  btn.disabled = true;
  btn.textContent = 'Submitting…';

  /* ─── BUILD YOUR PAYLOAD ────────────────────────────────────────
     Keys here must match the data.XXXXX keys in your Apps Script.
     Change these keys for each different form page.
  ─────────────────────────────────────────────────────────────── */
  const ref     = generateRef();
  const payload = {
    ref,
    timestamp:         new Date().toLocaleString('en-GB', { timeZone: 'Asia/Kathmandu' }),

    // ↓ Page-specific fields — change these per form
    requestType:       getRadioValue('req_type'),
    itemType:          getRadioValue('item_type'),
    itemDescription:   val('itemDescription'),
    partCode:          val('partCode'),
    quantity:          val('quantity'),
    installationNeeded:getRadioValue('install'),
    vehicleBrand:      val('vehicleBrand'),
    vehicleModel:      val('vehicleModel'),
    variant:           val('variant'),
    year:              val('year'),
    fuelType:          getRadioValue('fuel'),
    fullName:          name,
    mobile:            mobile,
    deliveryAddress:   val('deliveryAddress'),
    email:             val('email'),
    deliveryMethod:    getRadioValue('delivery'),
  };

  try {
    await submitToSheets(payload);
    // Show success
    btn.textContent = 'Request submitted — we\'ll be in touch shortly.';
    btn.style.background = 'var(--green-d)';
    // Optional: show ref number to user
    console.log('Reference:', ref);
  } catch(e) {
    btn.disabled = false;
    btn.textContent = 'Submit Request';
    console.error('Submission error:', e);
  }
}

/* ─── HELPERS — same on every page, no changes needed ─── */
function val(id) {
  return (document.getElementById(id)?.value || '').trim();
}
function getRadioValue(name) {
  return document.querySelector(`input[name="${name}"]:checked`)?.value || '';
}
What changes between pages: Only the SHEETS_URL at the top, and the payload object keys. The submitToSheets(), generateRef(), val(), and getRadioValue() helpers are identical on every page.

Required — add id="" to every form input
!
Every input needs an id attribute — the JS reads values by id
Radio buttons use the name attribute — those are already set

The current form pages use class="form-input" but many inputs are missing id="". The val(id) helper reads by id. You need to add an id to each input you want captured.

HTML — before and after 
<!-- BEFORE (no id — JS cannot read this) -->
<input class="form-input" type="text" placeholder="Your full name">

<!-- AFTER (id added — JS can read this with val('fullName')) -->
<input id="fullName" class="form-input" type="text" placeholder="Your full name">

<!-- SELECT inputs same pattern -->
<select id="year" class="form-select">...</select>

<!-- TEXTAREA same pattern -->
<textarea id="itemDescription" class="form-textarea"></textarea>

<!-- RADIO — use name="" not id (already set in existing forms) -->
<label class="radio-option">
  <input type="radio" name="fuel" value="petrol">Petrol
</label>
<!-- Read with: getRadioValue('fuel') → 'petrol' -->

Quick Reference
Full checklist per new page
1. Create new Google Sheet with headers
2. Paste Apps Script, update appendRow fields
3. Deploy as Web App, copy URL
4. Add id="" to all form inputs
5. Paste JS block, update SHEETS_URL and payload keys
6. Change onsubmit on the <form> tag to handleSubmit(event)
Common mistakes
— Forgetting to redeploy after editing the script
— appendRow array order doesn't match sheet headers
— Input missing id="" so val() returns empty
— Access set to "Only me" instead of "Anyone"
— Old URL used after redeployment
Form tag — change onsubmit
Every form's <form> tag must have:

onsubmit="handleSubmit(event)"

Replace any existing onsubmit attribute with this. The function is defined in the JS block you paste.
Why no-cors?
The browser blocks reading responses from different origins. mode:'no-cors' lets the request go through but hides the response. The data still reaches the sheet — you just can't read the confirmation back. That's why success state is shown immediately after the fetch resolves.
